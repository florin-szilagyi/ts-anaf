# P1.1 — Package bootstrap & build tooling

> **Workstream:** P1.1 (Wave W0) from the master plan at `docs/superpowers/plans/2026-04-11-anaf-cli-phases.md`.
>
> **For agentic workers:** Use `superpowers:executing-plans` to work this task-by-task. Checkboxes are the unit of progress.

**Goal:** Bootstrap the `anaf-cli` package as a pnpm workspace sibling to the existing SDK, with lint/format/test/build scripts ready, and produce a runnable stub binary that prints its version.

**Architecture:** Add `cli/` at repo root. Convert the repo into a pnpm workspace so `cli/` can depend on the SDK at the root via `workspace:*`. The SDK layout at the root does not move. `cli/` has its own `package.json`, `tsconfig.json`, source, tests, and dist. esbuild bundles the CLI to a single CJS file for the P4 SEA pipeline; `tsc` is used for type-checking only.

**Tech stack (locked here — propagates to every downstream workstream):**

| Concern | Choice | Notes |
| --- | --- | --- |
| Package manager | `pnpm` workspaces | matches existing repo |
| CLI parser | `commander@^12` | canonical, lowest risk |
| YAML | `yaml@^2` (eemeli/yaml) | used by P3.1 |
| Schema validation | `zod@^3` | used by P2.1 actions + P3.1 manifest kinds |
| Bundler | `esbuild@^0.21` | single-file CJS for SEA |
| Dev runner | `tsx@^4` | fast TS execution in dev |
| Test runner | `jest@^29` + `ts-jest@^29` | matches SDK |
| Node engine | `>=20` | SEA requires recent Node |
| Lint/format | reuse root eslint + prettier | extend root `.eslintrc.js` |
| `auth login` browser | default **open**, `--no-open` disables | matches `gh`/`vercel` |
| Token storage | plain JSON file `chmod 0600` | keychain deferred |
| macOS code signing | deferred (v1 ships unsigned) | document Gatekeeper workaround in P4.4 docs |

### Task 7: Source — version module and bin entrypoint

**Files:**
- Create: `cli/src/version.ts`
- Create: `cli/src/bin/anaf-cli.ts`

- [ ] **Step 1: Write cli/src/version.ts**

```ts
// Read at build time via tsc's resolveJsonModule.
// The literal is the SoT for the CLI's user-visible version.
import pkg from '../package.json';

export const CLI_VERSION: string = (pkg as { version: string }).version;
export const CLI_NAME = 'anaf-cli';
```

Note: tsc will emit this import correctly because `resolveJsonModule` is on. At bundle time (P4.1) esbuild will inline the literal. If `tsc` complains about rootDir because `package.json` is outside `src`, switch to a generated file (see fallback in Step 2b below).

- [ ] **Step 2: Try to build and verify rootDir behavior**

Run:
```bash
cd cli && pnpm run build:tsc
```

If tsc errors with `File '../package.json' is not under 'rootDir'`, apply Step 2b fallback.

- [ ] **Step 2b (fallback, only if Step 2 fails): Generate version.ts from package.json at build time**

Replace `cli/src/version.ts` with:
```ts
// This file is auto-generated by scripts/gen-version.mjs before tsc. Do not edit by hand.
export const CLI_VERSION = '0.0.0';
export const CLI_NAME = 'anaf-cli';
```

Create `cli/scripts/gen-version.mjs`:
```js
import fs from 'node:fs';
import path from 'node:path';
import { fileURLToPath } from 'node:url';

const here = path.dirname(fileURLToPath(import.meta.url));
const pkgPath = path.join(here, '..', 'package.json');
const outPath = path.join(here, '..', 'src', 'version.ts');

const pkg = JSON.parse(fs.readFileSync(pkgPath, 'utf8'));
const body = `// AUTO-GENERATED by scripts/gen-version.mjs. Do not edit.
export const CLI_VERSION = ${JSON.stringify(pkg.version)};
export const CLI_NAME = 'anaf-cli';
`;
fs.writeFileSync(outPath, body);
console.log(\`wrote \${outPath}\`);
```

Update `cli/package.json` scripts:
```json
"build:tsc": "node scripts/gen-version.mjs && tsc -p tsconfig.build.json",
"dev": "node scripts/gen-version.mjs && tsx src/bin/anaf-cli.ts",
```

- [ ] **Step 3: Write cli/src/bin/anaf-cli.ts**

```ts
#!/usr/bin/env node
import { Command } from 'commander';
import { CLI_NAME, CLI_VERSION } from '../version';

const program = new Command();

program
  .name(CLI_NAME)
  .description('CLI for the ANAF e-Factura SDK')
  .version(CLI_VERSION, '-v, --version', 'print the CLI version');

// Full command tree is registered in P1.2. For P1.1, the bin only needs --version and --help.
program.parseAsync(process.argv).catch((err: unknown) => {
  const message = err instanceof Error ? err.message : String(err);
  process.stderr.write(`${CLI_NAME}: ${message}\n`);
  process.exit(1);
});
```

The shebang is critical — npm/pnpm adds exec permissions to `bin` targets at install, and Homebrew/SEA will rely on it as a Node script before SEA wraps it.

### Task 14: Write the smoke test

**Files:**
- Create: `cli/tests/bin.smoke.test.ts`

- [ ] **Step 1: Write the test**

```ts
import { spawnSync } from 'node:child_process';
import path from 'node:path';

const BIN = path.resolve(__dirname, '..', 'dist', 'bin', 'anaf-cli.js');

describe('anaf-cli bin smoke', () => {
  it('prints the version and exits 0', () => {
    const result = spawnSync(process.execPath, [BIN, '--version'], {
      encoding: 'utf8',
    });
    expect(result.status).toBe(0);
    expect(result.stdout.trim()).toMatch(/^\d+\.\d+\.\d+$/);
  });

  it('prints help and exits 0', () => {
    const result = spawnSync(process.execPath, [BIN, '--help'], {
      encoding: 'utf8',
    });
    expect(result.status).toBe(0);
    expect(result.stdout).toContain('Usage: anaf-cli');
  });
});
```

Note: this test requires a prior build. Jest's smoke test script runs `build` separately (we don't couple build into jest globals to keep unit test runs fast).

## Out of scope for P1.1 (explicitly)

- No command handlers beyond `--version`/`--help` (P1.2+ territory).
- No XDG state paths, no token persistence (P1.4).
- No browser-open helper (P1.5).
- No lookup, no UBL (P1.7, P2.2).
- No SEA artifacts (P4.2).

- `pnpm install` succeeds at the repo root and wires `cli` as a workspace.
- `pnpm --filter anaf-cli run build` emits `cli/dist/bin/anaf-cli.js`.
- `node cli/dist/bin/anaf-cli.js --version` prints a semver string and exits 0.
- `pnpm --filter anaf-cli run build:bundle` emits a runnable `cli/dist/bin/anaf-cli.cjs`.
- `pnpm --filter anaf-cli run test:smoke` passes.
- `pnpm --filter anaf-cli run lint` has no errors.
- One commit on `feat/anaf-cli` captures all of the above.

## Handoff notes for downstream workstreams (W1)

When W1 picks up, these exports / paths are stable and can be relied on:

- CLI parser entry: `cli/src/bin/anaf-cli.ts`, which constructs a `commander.Command` named `program`.
- Version module: `cli/src/version.ts` → `CLI_VERSION`, `CLI_NAME`.
- The root `program` from `cli/src/bin/anaf-cli.ts` is the hook point for P1.2 to register `auth`, `ctx`, `efactura`, `lookup`, `ubl`, `run`, `schema` command groups. P1.2 should extract `buildProgram()` into `cli/src/commands/buildProgram.ts` and have the bin call it.
